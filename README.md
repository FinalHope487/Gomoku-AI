# Gomoku AI Benchmark 專案

這是一個用於測試與評估五子棋（Gomoku）AI 程式效能的作業。
包含一個自製測試框架，可以同時對多個 AI 版本進行對戰測試，並統計勝率、執行時間以及作業中定義的評分。

## 專案架構

- `benchmark.py`: 自動化測試主程式。使用subprocess並行測試，並產生詳細的效能統計報告。
- `hw1/referee.py`: 五子棋裁判程式。負責執行單場比賽，處理棋局邏輯、計時與判定勝負。
- `hw1/common.py`: 存放棋盤邏輯、移動驗證與勝負判定等共用函式。
- `hw1/`: 包含範例黑子引擎 `engine_minimax.py` 及根據現有資訊再現的黑子引擎 `engine_black2_var1.py` 等。
- `hw1_11427234_v*.py`: 白子程式。

## 快速開始

### 環境需求

- Python 3.x

### 執行測試

在專案根目錄執行以下指令：

```bash
python benchmark.py
```

## 測試功能

### 1. 多版本對戰

您可以在 `benchmark.py` 中的 `black_engines` (黑子) 與 `white_engines` (白子) 列表定義多個 AI 版本，程式會自動讓這些版本與 `black_engines` 中的對手進行循環賽。

### 2. 詳細統計報告

測試結束後，系統會印出：

- **總結數據**：總對局數、各方勝場、平手次數與錯誤次數。
- **白子程式個別統計**：
  - 勝敗紀錄（勝/敗/平）
  - 總得分 (Total Score)
  - 平均得分 (Avg Score)
- **單場對局資訊**：包含每步棋的耗時（最小/平均/最大值）。

### 3. 計分機制

根據裁判程式的規則，白子的得分計算方式如下：

- **獲勝**：根據剩餘步數與平均耗時給予高分 (224分~117分)。
- **平手**：給予基礎分數並扣除耗時 (117分~112分)。
- **落敗**：根據走子步數給予基礎分並扣除耗時 (112分~0分)。

### 4. 高分策略

已知最終測試會使用 4 份黑子程式評分並以加總後分數排名

假設 3 勝 1 負，勝場分數皆為 224 敗場分數 20 加總後為 692，平均僅有 173 分

可知只要輸了一場，總分數就會被大砍，因此白子程式設計應為在不要輸的前提下，爭取高勝率為目標

## 日誌記錄

測試結果都會自動記錄在 `benchmark_results.log` 中。

## 實際測試

### 黑子程式

- **engine_minimax.py**: Negamax 搭配 Alpha-Beta 剪枝。固定搜尋深度為 2，候選著法上限 12。
- **engine_black2_var1.py**: Negamax 搭配 Alpha-Beta 剪枝。搜尋深度提升至 3，但候選著法壓縮至 8 以維持效能。防守權重設為 1.0。
- **engine_black3_var2 / black4_aggro**: 在搜尋過程中給予進攻模式（如連三、連四）更高的權重倍率，強迫白子必須進行精準防禦。

### 對戰結果

測試中僅有 v9 和 v22 能獲得全勝，v9 整體步數又較 v22 更少一些 (約 5~7 步)，且 v22 受效能影響波動較大，在稍高負載下就會開始輸場，決定以微調參數後的 v9 作為最終提交版本。
